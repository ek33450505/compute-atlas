You have two responsibilities this session, both scoped to {{STATE}} for the Compute Atlas
dataset (a source-cited public tracker). Do both; combine the results into one output array.

=== Responsibility 1: discover net-new facilities ===

Research net-new AI data center, crypto-mining, or dedicated power-generation facilities in
{{STATE}} that are not already well-known/obviously duplicated in public trackers.

Scope for this responsibility:
- Only facilities in {{STATE}} not already well-known/obviously duplicated in public trackers.
- Only facilityType one of: data_center, crypto_mining, power_generation.
- Every fact must be traceable to a real, checkable source (press release, permit filing,
  ISO interconnection queue, OSM, subsidy/incentive filing, or other public filing).
- Do not fabricate coordinates, capacity, operator names, or dates. If a field is unknown,
  omit it rather than guessing.

For each new facility you find:
1. Research it using real web sources. Cite every source you use with a URL.
2. Build a JSON object matching the Facility schema in lib/schema.ts — see the
   discriminated union on `facilityType` (data_center | crypto_mining | power_generation),
   the shared base fields (id, name, operator, status, confidence, location, sources, ...),
   and the type-specific fields for whichever facilityType applies.
3. `id` must be a lowercase kebab-slug unique to this facility (e.g. "operator-name-city-state").
4. `sources` must be a non-empty array of `{ url, label, retrievedAt, kind }` objects — the
   staging queue rejects any candidate without at least one cited source.
5. Attach a `provenance` object alongside the facility payload:
   `{ sources: string[] (the same URLs), confidence, discoveredBy: "discovery-pipeline",
      note: "brief summary of why this is a credible new candidate" }`.

=== Responsibility 2: re-check existing facilities for genuine status changes ===

Below is a compact projection of facilities Compute Atlas already tracks in {{STATE}}, one
line per facility: `id | name | operator | status | <latest status date> | <primary source url>`.

{{EXISTING_FACILITIES}}

For each listed facility, check whether real-world sources show a genuine status change since
its listed status/date (e.g. announced/proposed -> under_construction -> operational, or a
cancellation). This is a status-refresh pass only — do not attempt to update or verify
capacity, type, location, operator, or any other field.

Rules for this responsibility:
- Only emit an entry for a facility if you find a NEW, real, citable source that corroborates
  an actual status change. Never infer, guess, or fabricate a change. If you cannot find a
  genuine sourced change for a facility, do NOT emit an entry for it — omit it entirely from
  the output. Do not "refresh" a facility just because it appeared in the list.
- When you do find a genuine status change, DO NOT reconstruct the full facility document —
  the compact projection above does not give you enough of the record to rebuild it faithfully
  (you cannot see the full `sources` array, location, capacity, or prior `statusHistory`), and
  guessing at those fields silently corrupts the record. Instead, emit a compact status-
  transition intent:
  `{ "statusUpdate": { "targetFacilityId": "<the exact id from the projection line>",
  "status": "<new source-backed status>", "date": "<today, YYYY-MM-DD>",
  "note": "<brief why>", "sources": [ <the NEW corroborating source object(s):
  {url,label,publisher,retrievedAt,kind}> ] }, "provenance": { "sources": [<same urls>],
  "confidence": "...", "discoveredBy": "discovery-pipeline", "note": "brief summary of the
  sourced status change" } }`
  - `targetFacilityId` MUST equal the id from the projection line exactly — do not invent a
    new id for an existing facility.
  - `sources` in the intent is ONLY the new corroborating source(s) you found this session —
    never restate or guess at the facility's prior sources. The server appends your new
    source(s) and a new `statusHistory` entry automatically and preserves everything else
    (name, operator, location, prior sources, prior history) unchanged. You must not restate
    fields you cannot see, and you must not emit a full facility document for this
    responsibility.

=== Output requirements — read carefully ===
